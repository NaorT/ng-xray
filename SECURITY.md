# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in ng-xray, please report it responsibly using [GitHub's private vulnerability reporting](https://github.com/NaorT/ng-xray/security/advisories/new).

**Do NOT** open a public issue for security vulnerabilities.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Affected versions
- Suggested fix (if any)

### Response timeline

- Acknowledgment within 48 hours
- Fix or mitigation plan within 7 days
- Public disclosure after fix is released

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Security Model

See the [Security Model section in README](README.md#security-model) for details on ng-xray's trust boundary between static-only and exec-based analyzers.
